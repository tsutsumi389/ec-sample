# デザイン性 評価基準（実画面スクリーンショットベース・100点満点）

> **本書は「デザイン性（美的品質・ビジュアルデザインの洗練度）」の評価軸であり、`docs/UIUX_VISUAL_EVALUATION.md` が扱う「UI/UX（使いやすさ・機能面）」の評価とは独立した別軸である。**
> UI/UX軸が「壊れていない・使える・分かる」を判定するのに対し、本軸は「造形として洗練されているか・美しいか・固有のプロダクトに見えるか」を判定する。
> UI/UX軸で採点済みの観点（エラー表示の分かりやすさ、導線の明確さ、コントラスト充足、フォーカスリング、タッチターゲットサイズ、レスポンシブの破綻有無等）は本書の採点対象に**含めない**。両者は独立に採点し、点数を合算しない。

対象: 本アプリの実際の画面（ブラウザでレンダリングされた状態）。ゲスト / 一般ユーザー / 管理者の3状態、デスクトップ（1280px）とモバイル（375px）の2ビューポートで撮影したスクリーンショットを主対象とする。最低限、以下のページ・状態をカバーすること。

- ゲスト: トップ（`/`）、商品詳細（`/products/[id]`）、カート、ログイン、会員登録
- 一般ユーザー: ログイン後トップ、カート（商品あり）、注文履歴（`/orders`）、注文詳細（`/orders/[id]`）
- 管理者: ダッシュボード（`/admin`）、商品管理（`/admin/products`・作成/編集モーダル含む）、注文管理（`/admin/orders`）、ユーザー管理（`/admin/users`）
- **状態撮影（本軸固有の必須要件）**: 主要CTA（カートに追加・注文を確定する・ログイン）と商品カードについて、hover 時・disabled 時（送信中／在庫切れ）のスクリーンショットを追加撮影すること。撮影がない場合、該当項目（4-6）はコード確認で判定してよいが**配点の50%を上限**とする。

合格ライン: **80点以上**。

評価方法: 上記スクリーンショットの目視確認を主とする「実画面ベース」の評価。拡大・計測（余白のピクセル比、色のスウォッチ切り出し、全ページ俯瞰での並置比較）を積極的に用いる。`transition` 指定の有無など静的画像で判定できない点のみ、該当ソースコードで補助確認する（表中に「コードで補助確認」と明記）。

## 採点ルール

- 各項目は 0〜配点の整数で採点する。部分的に満たす場合は按分してよい。
- **根拠必須**: 採点にはスクリーンショットのファイル名（例: `guest-products--desktop.png`）を必ず引用すること。コードで補助確認する場合はファイルパス・行番号を引用する。根拠を示せない加点は禁止。
- **疑わしきは減点（辛口採点）**: スクリーンショットから充足を明確に確認できない場合は「満たしていない」として扱う。好意的な推測による加点は禁止。判定に迷う場合は低い方の点を付ける。
- **「一部のページ・状態のみ対応」は配点の50%を上限とする**。例えば「ストア側は統一されているが管理画面では崩れている」「デスクトップは美しいがモバイルは縮小しただけ」のケースは、その項目の配点の半分を超えて加点してはならない。
- 撮影対象に含まれないページ・状態がある場合は、確認できた範囲内で採点し、評価コメントに「未確認: ◯◯」を明記する。
- 計測・横断比較を要する項目（1-1、3-1、3-3 等）は、検出した値や色の一覧を評価コメントに添付すること。
- UI/UX軸（`UIUX_VISUAL_EVALUATION.md`）で既に減点・加点された事象を本軸で二重に採点しない。境界が曖昧な場合は「機能として使えるか＝UI/UX軸」「造形として美しいか＝本軸」で振り分ける。
