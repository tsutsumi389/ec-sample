03技術スタック

レイヤー技術役割・補足
フロントエンドNext.js 14（App Router）/ React / TypeScript / Tailwind CSS全ページ 'use client' の SPA 的構成。共通レイアウト＋認証 Context。
バックエンドFastAPI / SQLAlchemy 2.0（Mapped/mapped_column）/ Pydantic全ルートは /api プレフィックス配下。依存注入で認証・DB セッションを供給。
データベースPostgreSQL 16 + pgvector 拡張17 モデル。ベクトル検索（コサイン距離）に pgvector を利用。
AI / 埋め込みOllama（埋め込み + チャット）埋め込み embeddinggemma（768 次元）、チャットモデルで理由生成。
認証JWT（HS256, PyJWT）/ passlib bcryptBearer トークン、有効期限 24h、パスワードは bcrypt ハッシュ。
実行基盤Docker / docker-compose / Makefile開発操作は Makefile に集約（make up / make reset 等）。

3.1docker-compose サービス構成

db

PostgreSQL 16 + pgvector。DATABASE_URL のデフォルトは postgresql://ec:ecpass@db:5432/ecdb。

backend

FastAPI（EC Sample API）。起動時 lifespan でテーブル生成・シード投入・埋め込み同期を実行。CORS は http://localhost:3000 のみ許可。

frontend

Next.js。NEXT_PUBLIC_API_URL（既定 http://localhost:8000）+ /api に通信。

外部連携

Ollama は OLLAMA_BASE_URL（既定 http://host.docker.internal:11434）で参照する。未起動・未 pull でもアプリ起動と API 応答は止まらない（フォールバック動作、詳細は §8）。
