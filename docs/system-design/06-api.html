06API 設計

バックエンドの全ルートは /api プレフィックス配下に一括登録される（main.py）。認証要件は none 認証不要 / user ログイン必須 / admin 管理者のみ、で示す。

6.1認証（/api/auth）

MethodPath認証概要
POST/api/auth/registernone新規登録。メール重複は 400。bcrypt ハッシュ、role=user で作成、201 で UserOut
POST/api/auth/loginnone認証。不一致/不在は一律 401。成功で 24h トークン（Token）を発行
GET/api/auth/meuser認証中ユーザー自身の情報（UserOut）
PUT/api/auth/meusername を更新して UserOut を返す
PUT/api/auth/me/passworduserパスワード変更。current_password 検証（不一致 400）、既存トークン失効、204

6.2商品・カテゴリ・レビュー・お気に入り

MethodPath認証概要
GET/api/products任意商品一覧。search / category_id / sort（newest / price_asc / price_desc / rating / recommended）/ min_price / max_price / page（既定 1）/ limit（既定 12, 1-100）。LISTED_STATUSES のみ、平均評価・件数・total 付き。search はハイブリッド検索: name ILIKE ∪ pgvector 意味的候補（クエリ埋め込みのコサイン近傍・動的カットオフ）。検索中に sort 未指定なら関連度順（ILIKE 一致優先 → 意味的距離）で並び、sort 明示時はセマンティックはフィルタとしてのみ作用。sort=recommended はログイン時プロフィール近傍、非ログイン・不能時は人気順（任意認証・詳細は §8.6）
GET/api/products/suggestnone検索サジェスト。q（2 文字未満は DB を引かず即空返し）/ limit（既定 8, 1-20）。LISTED 商品名への ILIKE のみの軽量クエリで、キーワード候補 suggestions（同名は畳む）と商品ダイレクト候補 products（最大 3 件: id / name / image_url / price / sale_price / effective_price）を同じ関連度順（strpos → 名前長 → 名前）で返す。入力中の高頻度呼び出しに耐えるため埋め込みは使わない。詳細は docs/design/search-uiux.html
GET/api/products/{id}none商品詳細。viewable でなければ 404。平均評価・件数付き
POST/api/products/{id}/view任意閲覧記録（パーソナライズのシグナル収集）。ゲストは no-op で 204、viewable でなければ 404。INSERT … ON CONFLICT DO UPDATE で view_count+1／viewed_at 更新の upsert（並行初回閲覧の競合を吸収）。204
GET/api/products/{id}/relatednone関連商品。同カテゴリの LISTED を自身除外・id 昇順で最大 4 件
GET/api/products/{id}/recommendationsnone類似おすすめ。pgvector コサイン近傍、無ければ同カテゴリフォールバック。limit 既定 4（1-20）
GET/api/products/{id}/reviewsnoneレビュー一覧。投稿者名付き、created_at 降順・id 降順
POST/api/products/{id}/reviewsuserレビュー投稿。購入者（cancelled 以外の注文で購入）のみ、1 商品 1 件。201
GET/api/products/{id}/questionsnone商品Q&A 一覧。新しい順・最大 100 件、質問者名付き。viewable でなければ 404
POST/api/products/{id}/questionsuser質問投稿（1〜300 字）。その商品の説明文＋レビューだけを根拠に AI 回答を同期生成して永続化。Ollama 失敗時も source="fallback" の定型文で必ず 201（§8.8）
GET/api/categoriesnoneカテゴリ一覧。名前昇順で全件
GET/api/wishlistuser自分のお気に入り一覧。created_at 降順・id 降順
POST/api/wishlist/itemsuserお気に入り追加。viewable でない商品は 404、重複時は既存返却（冪等）。201
DELETE/api/wishlist/items/{product_id}userお気に入り削除。存在しなくても 204（冪等）

6.3カート・注文・住所帳・クーポン

MethodPath認証概要
GET/api/cartuserカート内容と合計金額を取得
POST/api/cart/itemsuserカート追加。is_viewable かつ status==on_sale、合算後数量が在庫以下を検証。既存行は数量合算。201

DELETE/api/cart/items/{item_id}user明細削除しカート全体を返す。自分の明細のみ
POST/api/ordersuser注文確定。在庫ロック→在庫チェック→スナップショット→在庫減算→クーポン適用→カート削除を 1 トランザクション。201
GET/api/ordersuser自分の注文一覧。created_at 降順・id 降順
GET/api/orders/{order_id}user自分の注文詳細。他人の注文は 404
POST/api/orders/{order_id}/canceluserキャンセル。pending / paid のみ許可。在庫を数量分戻して status=cancelled
GET/api/addressesuser住所一覧。is_default 降順・id 降順
POST/api/addressesuser住所作成。is_default が真なら他のデフォルトを解除。201

DELETE/api/addresses/{address_id}user住所削除。204。自分のみ
POST/api/coupons/validateusercode と subtotal を受け取り有効性・割引額・メッセージを返す。認証必須

6.4レコメンド・ホーム

MethodPath認証概要
GET/api/recommendations/home任意単一のおすすめ枠。ログイン時は ready かつ profile_hash 一致のキャッシュを rank 順で返却、無ければ人気順を即返しつつ生成をスケジュール。未ログインは人気順のみ。source で llm / fallback を区別
GET/api/home任意ホーム画面のレーン構成（Netflix 型）。recently_viewed_ids（"12,7,3" 形式・先頭 10 件）/ max_lanes（既定 8, 1-12）を受け取り、複数レーンを stage-wise 貪欲法で並べた HomeOut（source = personalized / popular ＋ sections[]）を返す。LLM キャッシュ陳腐化時は /recommendations/home と同流儀で再生成をスケジュール（レスポンスはブロックしない）。多層フォールバックで常に 200。詳細は §8.7

6.6AI アシスタント（/api/assistant）

チャット型ショッピングアシスタント。認証は 任意（get_current_user_optional）で未ログインでもゲスト会話として利用できる。Ollama 失敗時も source="fallback" で 200 応答し、この API がユーザーに 500 を返すことはない。

MethodPath認証概要
POST/api/assistant/chat任意相談文を受け取り応答＋提案商品カードを返す。conversation_id=null なら新規会話作成、既存 UUID は所有チェック（他人/不在は 404）。message は 1〜500 字、会話上限 50 メッセージ（超過は 400）。source で llm / fallback を区別
GET/api/assistant/conversations/{id}/messages任意ウィジェット再オープン時の履歴復元。所有チェック後に id 昇順で全メッセージを返す。assistant 行は product_ids から商品を引き直し、LISTED_STATUSES を再確認して非公開化された商品はカードから落とす
認可

ログインユーザーの会話（user_id あり）は本人のみアクセス可、それ以外には存在を秘匿して 404。ゲスト会話（user_id NULL）は UUID を知っていることが認可（サンプルとして許容、詳細は §11）。ゲスト会話中にログインすると chat 側で user_id を紐付けて引き継ぐ。
