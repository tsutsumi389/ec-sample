04アーキテクチャ全体像

ブラウザ（Next.js SPA）→ FastAPI（/api）→ PostgreSQL の 3 層に、AI レコメンド用の Ollama を側面連携させた構成である。フロントは fetch ラッパ経由で JWT を自動付与し、バックエンドは依存注入で認証・DB を供給する。埋め込み生成・LLM 推論は同期パスに載せず、起動時スレッドや BackgroundTasks で非同期に走らせる。

Client
Next.js 14 SPA店頭画面 / 管理画面（'use client'）
AuthProviderReact Context・JWT を localStorage 保持
lib/api.tsfetch ラッパ・Authorization: Bearer 自動付与
▼   HTTP / JSON（NEXT_PUBLIC_API_URL + /api）  ▼
Application
FastAPI ルーター群auth / products / cart / orders / addresses / coupons / wishlist / recommendations / assistant / home / admin
依存注入get_current_user / _optional / get_current_admin / get_db

耐障害性

pgvector 拡張が使えない環境では product_embeddings を作成対象から除外して起動継続。Ollama が未稼働でも埋め込み・SID 再割り当て・LLM 生成の失敗はすべて握られ、人気順フォールバックで API は常に応答する。
