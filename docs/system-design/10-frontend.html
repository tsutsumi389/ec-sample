10フロントエンド設計

Next.js 14 App Router 構成。全ページが 'use client' のクライアントコンポーネントで、app/layout.tsx が AuthProvider・Header・Footer を全ページに敷き、認証状態を React Context で一元管理する。データ取得は各ページの useEffect＋fetch で行う SPA 的構成。

10.2認証コンテキストと API クライアント

AuthProvider / useAuth

lib/auth-context.tsx。user / loading と login / register / logout を提供。初回にトークンがあれば /auth/me で復元。ログイン変化を依存に持つコンポーネントは再取得する。

lib/api.ts

fetch ラッパ（get/post/put/delete）。JWT を localStorage（キー token）から Authorization: Bearer に自動付与、401 でトークン破棄、204/空ボディ対応、英語 detail を日本語化。

状態の集約

商品 status は lib/productStatus.ts（PRODUCT_STATUS_META / ADMIN_SELECTABLE_STATUSES）、注文 status は lib/order-status.ts にラベル・色・選択肢を集約。

10.3UI 規律

  • 保護ページ（cart/orders/account/wishlist）は !authLoading && !user で /login?redirect=… へ誘導。管理ページは AdminLayout が role で弾く
  • 価格の配色規律: 金額は常に text-gray-900・font-semibold で統一し赤字を使わない。ProductPrice はセール時のみ定価打消し線＋NN%OFF バッジを併記
  • brand 塗り（btnPrimary）は最重要 CTA 限定。共通見た目は Badge / Price / StockLabel / RatingStars 等と buttonStyles.ts 定数で統一
  • モーダルは role=dialog + aria-modal、自前フォーカストラップと Esc クローズ。画像は素の <img> で onError 時 /no-image.svg にフォールバック
  • 各ページはメタデータ API を使えないため、RootLayout が usePathname でパス別に document.title を設定

10.4AI アシスタントウィジェット

app/layout.tsx が <AssistantWidget /> を全ページ末尾に敷く（AuthProvider 内）。右下フローティングボタンでパネルを開閉する常駐ウィジェットで、components/assistant/ 配下の 3 コンポーネントで構成する。

AssistantWidget

フローティングボタン＋パネル開閉（'use client'）。usePathname で /admin 配下なら null を返し管理画面では非表示。モバイルは開いている間ボタンを隠す。

AssistantPanel

メッセージリスト・入力欄・送信・ローディング。モバイルは全画面、PC は 380px 幅パネル。会話 ID を localStorage に保持し、再オープン時に履歴 API で復元（無効=404 なら ID を破棄）。「新しい会話」ボタンは ID を破棄して画面リセット。

AssistantProductCard

チャット内のコンパクト商品カード。画像・商品名・ProductPrice を再利用し、クリックで /products/[id] へ遷移（パネルは開いたまま）。

  • 会話継続: conversation_id を localStorage に保存し、パネル再オープン時に api.assistant.messages() で履歴復元。未ログインでも端末単位で会話が続く
  • 多重送信防止: 送信中はスピナー＋入力無効化。応答まで最大 60 秒待つ旨のプレースホルダを表示（同期パスのレイテンシ対策）
  • 初回オープンはウェルカムメッセージ（クライアント固定文言、API は呼ばない）＋サジェスト chips（タップで入力欄に挿入、自動送信はしない）
  • source="fallback" の応答には「キーワード検索の結果です」の注記を添える
  • lib/api.ts に api.assistant.chat() / api.assistant.messages() を追加（既存 fetch ラッパー踏襲、未ログインでも呼べる）

10.5ホーム画面のレーン描画

トップ（app/page.tsx）のヒーロー＋単一おすすめ枠を、GET /api/home を 1 リクエストだけ叩いて返ってきた sections[] を layout に応じて描き分ける構成に刷新した。認証トークンが localStorage 保持のため Server Component 化できず、レーンごとの個別 fetch によるウォーターフォールを避けるべく /home に集約している。

HomeSections

ホームのレーン群のコンテナ（'use client'）。useAuth の user を依存に /home を取得し、ログイン状態が変われば取り直す。ゲストは getRecentlyViewedIds()（localStorage）を recently_viewed_ids に載せてパーソナライズ。取得失敗時は BrandHero だけ出して画面を壊さない。

HomeBillboard

最上部のビルボード（layout: hero）。商品 1 件＋理由文（reason）を大きく訴求。暗いブランド背景で ProductPrice（text-gray-900 固定）が読めないため価格だけ白文字で自前描画（打ち消し線つき定価の併記ルールは踏襲）。

ProductLane

横スクロールレーン（lane / ranked）。カルーセルライブラリを使わず素の CSS scroll snap で実現。端検知で矢印・フェードを出し分け、キーボード操作対応。ranked は順位番号を大きく併記（社会的証明の可視化）。

BrandHero

ビルボードを出せないとき（商品無し・/home 取得失敗・コールドスタートで hero レーン無し）のフォールバック。旧トップの装飾ヒーロー（ケトル・湯呑み・植物の SVG）を切り出したもの。

  • 読み込み中は BillboardSkeleton＋LaneSkeleton で高さを予約し段差を防ぐ。sections が空なら EmptyState を表示
  • hero は先頭 1 本のみ採用（契約上も billboard は 1 本）。それ以外を ProductLane に振り分け、key は section.key（billboard / top10 / byw:{id} / category:{id} など一意）を使う
  • 「最近見た商品」はレーン（recently_viewed）が担うためトップ末尾の RecentlyViewed は撤去（同コンポーネントは商品詳細ページで継続利用）。型は lib/types.ts の HomeResponse / HomeSection / HomeSectionLayout

10.6検索窓（SearchBox・サジェスト）

components/SearchBox.tsx。Header に PC 常時表示＋モバイル開閉バーの 2 箇所で配置され、WAI-ARIA combobox パターン（role="combobox" / listbox / option、aria-activedescendant）に準拠する。確定で /?search=… へ遷移しフルのハイブリッド検索に渡す。詳細設計は docs/design/search-uiux.html。

  • 3 種のサジェスト: 入力 2 文字未満は localStorage の検索履歴（lib/searchHistory.ts、最大 5 件・個別削除・全消去）、2 文字以上はデバウンス 250ms＋AbortController で GET /products/suggest を叩き、キーワード候補（一致部分を太字強調）と商品ダイレクト候補（サムネイル＋実売価格、クリックで /products/{id} へ直行）を表示
  • 選択肢のフラット配列モデル: 履歴・キーワード・商品を Option = { kind: 'history' | 'keyword' | 'product' } の種別付き 1 次元配列に正規化し、↑↓ / Enter / aria-activedescendant が種別を問わず一貫動作。セクション見出しは配列に含めず role="presentation" で描画（キーボードで止まらない）
  • IME ガード: e.nativeEvent.isComposing || e.keyCode === 229（Safari の compositionend 後 Enter 対策）を判定し、変換確定の Enter は preventDefault で form の暗黙送信ごと止める。変換中は矢印・Esc も無効化（日本語入力での誤検索防止）
  • URL 同期: useSearchParams で ?search= を検索欄へ反映（直接アクセス・リロード・ブラウザ戻る対応）。タイピングを潰さないよう「パラメータが実際に変わったときだけ」反映し、URL 由来の反映ではドロップダウンを開かない。SearchBox が useSearchParams を使うため Header 内 2 箇所を Suspense でラップ（無いと next build が落ちる）
  • 入力があるときだけクリア（×）ボタンを表示。候補クリックは blur より先に処理するため mousedown で拾い preventDefault でフォーカスを維持（既存パターン踏襲）
