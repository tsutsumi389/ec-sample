02システム概要

Hibino は Docker 上で動作する EC サイトのサンプルである。店頭（一般ユーザー）と管理の 2 系統の画面を持ち、商品閲覧から購入、注文管理、AI による生成的レコメンドまでを一通り備える。商品の可視性・購入可否は Product.status（6 状態の状態機械）を唯一の源として導出し、実売価格は effective_price（セール価格優先）に一元化されている点が設計の背骨となっている。

2.1主要機能サマリ

ホーム（レーン構成）

Netflix 型の複数レーンでホームを構成。1 レーン = 1 アルゴリズム（ビルボード / 買い忘れ / あなたへ / 売れ筋 Top10 / カテゴリ / セール / 新着ほか）を stage-wise 貪欲法で並べ、ゲストも閲覧履歴でパーソナライズ。

商品カタログ

セマンティック（ハイブリッド）検索・カテゴリ絞り込み・価格帯フィルタ・並び替え・ページング付き一覧、商品詳細、関連商品、平均評価表示。検索窓は履歴・キーワード・商品直行の 3 種サジェスト付き（§10.6）。

レビュー

購入者限定の投稿（1 ユーザー 1 商品 1 件）、投稿者名付き一覧、平均評価集計。

お気に入り

ユーザー単位で商品を登録・一覧・削除。追加/削除とも冪等操作。

カート・注文

カート操作、在庫ロック付き注文確定、注文履歴・詳細・キャンセル（在庫戻し）。

住所帳・クーポン

複数配送先（デフォルト 1 件保証）、クーポン検証と注文時適用（percent / fixed）。

AI レコメンド

ユーザー行動から Ollama × pgvector × セマンティック ID で生成的におすすめと理由を提示。

AI アシスタント

全ページ常駐のチャット型店員 AI。自然文の相談にカタログ商品を根拠付きで提案。未ログイン可、Ollama 不在でもフォールバック応答。

商品Q&A（購入前 AI 回答）

商品ページで購入検討者の質問に、その商品の説明文とレビューだけを根拠に AI が即答・蓄積。回答はスナップショットとして公開され、未ログインでも閲覧可（社会的証明）。

認証・認可

JWT によるステートレス認証、一般ユーザーと管理者の分離、パスワード変更時の全セッション失効。

管理機能

商品/カテゴリ/クーポン CRUD、全注文管理・ステータス更新、ユーザー一覧、レコメンド再構築。
