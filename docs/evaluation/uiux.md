# UI/UX 評価基準（100点満点）

対象: `frontend/` 配下の全ページ・コンポーネント。
合格ライン: **80点以上**。
評価方法: コードレビューによる静的評価。各項目は減点方式ではなく「満たしている度合い」で加点する。判定には必ずコード上の根拠（ファイル・行）を示すこと。

## 1. アクセシビリティ（20点）

| 項目 | 配点 | 判定基準 |
|---|---|---|
| セマンティックHTML | 4 | `main`/`nav`/`header`/`section` 等の適切な使用。div の乱用がない |
| 画像・アイコンの代替テキスト | 3 | `img` に意味のある `alt`。装飾的要素は `aria-hidden` |
| フォームのラベル関連付け | 4 | 全入力に `label htmlFor` または `aria-label` が関連付く |
| フォーカス可視性・キーボード操作 | 4 | フォーカスリングを消していない。クリック可能要素が `button`/`a` である（div+onClick でない） |
| ARIA・ステータス通知 | 3 | モーダルに `role="dialog"`/`aria-modal`、エラー表示に `role="alert"` 等 |
| コントラスト・文字サイズ | 2 | 淡すぎる文字色（gray-300 on white 等）や極端な小サイズがない |

## 2. フィードバック・状態表示（20点）

| 項目 | 配点 | 判定基準 |
|---|---|---|
| ローディング状態 | 5 | データ取得中にスピナー/スケルトン等の表示がある（全ページ） |
| エラー状態 | 5 | API 失敗時にユーザーが理解できるエラーメッセージを表示（silent fail がない） |
| 空状態（empty state） | 4 | 商品0件・カート空・注文履歴なし等に案内付きの空状態表示がある |
| 操作の成功フィードバック | 3 | カート追加・注文確定・保存等の成功が視覚的に伝わる（トースト等） |
| 多重送信防止・処理中表示 | 3 | 送信ボタンが処理中 disabled になり、状態がラベルでも伝わる |

## 3. レスポンシブ・レイアウト（15点）

| 項目 | 配点 | 判定基準 |
|---|---|---|
| ブレークポイント対応 | 5 | 主要ページのグリッド/レイアウトが sm/md/lg で破綻しない |
| モバイルナビゲーション | 4 | ヘッダーが狭い画面で崩れない（折返し・メニュー化等の対処がある） |
| テーブル・横長要素の対処 | 3 | 管理画面テーブル等に `overflow-x-auto` 等の横スクロール対処 |
| タッチターゲットサイズ | 3 | ボタン・リンクが十分な大きさ（極小のクリック領域がない） |

## 4. 一貫性・ビジュアルデザイン（15点）

| 項目 | 配点 | 判定基準 |
|---|---|---|
| ボタン・色のシステム化 | 5 | primary/danger 等の色使いが全ページで統一。同義の操作が同じ見た目 |
| 余白・タイポグラフィの一貫性 | 4 | 見出しレベル・余白スケールがページ間で揃っている |
| 金額・日付フォーマット統一 | 3 | 通貨表示（¥・カンマ区切り）と日付表示が全画面で同一形式 |
| 視覚的階層 | 3 | ページタイトル→セクション→本文の階層が明確 |

## 5. フォーム UX（15点）

| 項目 | 配点 | 判定基準 |
|---|---|---|
| バリデーションとエラー表示 | 5 | 必須・形式エラーがフィールド近傍に日本語で表示される |
| 入力タイプ・属性の適切さ | 4 | `type="email"`/`number`、`autoComplete`、`inputMode` 等の適切な指定 |
| 必須項目の明示 | 3 | 必須フィールドがラベル上で判別できる |
| 破壊的操作の確認 | 3 | 削除・キャンセル等に確認ステップがある |

## 6. ナビゲーション・情報設計（15点）

| 項目 | 配点 | 判定基準 |
|---|---|---|
| 現在地の把握 | 4 | アクティブなナビ項目の強調、ページタイトル（`metadata`/`h1`）の設定 |
| 戻る導線 | 4 | 詳細ページに一覧へ戻るリンク、注文完了後の導線がある |
| 404・未認証時の挙動 | 4 | 存在しないID・未ログインアクセスで適切な表示/リダイレクト |
| 主要動線の短さ | 3 | 商品発見→カート→購入が迷わず辿れる（CTA が明確） |

## 採点ルール

- 各項目は 0〜配点の整数で採点。部分的に満たす場合は按分する。
- 「一部のページのみ対応」は配点の半分を上限とする。
- 根拠のない加点は禁止。判定にはファイルパスを引用すること。
